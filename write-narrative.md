# Skill: Write Narrative Script

Use this skill every time you need a voiceover script for an episode.
This is a repeatable instruction. Follow it exactly.

-----

## When to Use This Skill

When you have:

- An approved episode outline row (hook + conflict + power moment + cliffhanger)
- A locked character pack

-----

## Input Required

1. Episode number
1. Episode outline (hook, conflict, power moment, cliffhanger)
1. Character name and key trait
1. Platform target: Hongguo (faster pace) or YouTube (slightly more breathing room)

-----

## Output Format

A voiceover script with:

- Word count: 200–250 words (English)
- Duration target: 60–90 seconds when read at natural pace
- Every sentence under 15 words
- No sentences that start with “In a world where…”

-----

## Structure Rules (Non-Negotiable)

**Seconds 0–3 (Hook):**
Start with ONE of these:

- A line of dialogue that reveals conflict
- A physical action that signals danger
- A question that implies a secret the audience doesn’t know yet
- A reversal of expectation (something familiar shown to be wrong)

No narration. No setting description. Start in the action.

**Seconds 4–30 (Rising Tension):**

- Establish who the protagonist is through action, not description
- Show the core injustice, threat, or challenge
- One power moment or reversal in this section

**Seconds 31–60 (Escalation):**

- The situation gets harder or more complex
- Protagonist makes a choice or reveals a hidden capability
- Pace increases (shorter sentences)

**Final 10 seconds (Cliffhanger):**

- End on an unanswered question, a new threat, or an identity reveal
- Never resolve the episode. Always leave one thing unresolved.
- Last line should work as a standalone hook if someone sees it out of context.

-----

## Quality Check Before Approving

Read it out loud. Ask:

- Does the first line make me want to hear the next line?
- Is there at least one moment where I feel something?
- Does it end before I expect it to?
- Could I cut 20% of the words and keep the meaning?

If the answer to any of these is no — revise before approving.

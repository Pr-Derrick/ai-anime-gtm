### scratchpad.md

```markdown
# Scratchpad — AI 动漫漫剧项目
# Scratchpad — AI Anime Drama Project

## 问题定义 / Problem Definition

**这个 workflow 要解决什么决策？**
**What decision does this workflow improve?**

中文：我需要知道每周应该做什么内容、发哪个平台、用什么节奏，

## Sad Story / 失败场景

中文：
我花了3天做了一集AI动漫，发到三个平台，
播放量加起来不到500。不知道哪里出了问题——

English:
I spent 3 days making one AI anime episode, 
posted it to three platforms, got less than 500 views combined.
No idea what went wrong — wrong genre? Wrong platform? 
Bad thumbnail? Wrong posting time?
No data, just guessing. Repeat the same mistake next week.

## Happy Story / 成功场景

中文：
每集发布后，我能在24小时内看到各平台的核心数据对比。
我知道：红果的末日题材比宫斗高3倍点击，

下周我知道该做什么。

English:
After each episode posts, I can see cross-platform metrics within 24 hours.
I know: apocalypse genre gets 3x more clicks than palace drama on Hongguo.

I know exactly what to make next week.
```

### Template C — 数据追踪表结构 / Data Tracking Schema

```sql
-- 平台发布记录 / Platform Post Log
CREATE TABLE posts (
  id INTEGER PRIMARY KEY,
  episode_number INTEGER,
  title_cn TEXT,
  title_en TEXT,
  genre TEXT,           -- apocalypse / cultivation / palace / other
  platform TEXT,        -- hongguo / douyin / youtube / x
  post_date DATE,
  thumbnail_version TEXT
);

-- 24小时数据 / 24-Hour Metrics
CREATE TABLE metrics_24h (
  post_id INTEGER,
  views INTEGER,
  completion_rate REAL,  -- 完播率
  click_rate REAL,       -- 点击率
  saves INTEGER,         -- 收藏数
  shares INTEGER,
  comments INTEGER,
  platform TEXT,
  collected_at TIMESTAMP
);

-- 周报汇总 / Weekly Summary
CREATE TABLE weekly_summary (
  week_start DATE,
  best_platform TEXT,
  best_genre TEXT,
  total_views INTEGER,
  avg_completion_rate REAL,
  top_episode INTEGER,
  insight TEXT           -- 本周关键结论 / Key insight this week
);
```

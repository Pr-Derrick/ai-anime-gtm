Scratchpad

Sad Story failure mode

Spent 3 days making one AI anime episode. Posted to three platforms. Under 500 views combined. No idea what went wrong — wrong genre, bad thumbnail, no hook, wrong time? No data. Just guessing. Same mistakes next week.

Happy Story success mode

After each publish, cross-platform report in 24 hours. I know which genre gets 3× higher CTR, which platform drives followers. I know exactly what to make next week. 3 episodes per week, under ¥500 each.

Minimum Viable Test

1 theme → complete 10-step pipeline → 1 published episode on Hongguo → 72-hour data collection → pass if completion rate ≥ 30% and no production errors

The Five Questions

What decision does this improve? — What content to make each week, which platform to prioritize, at what cadence. Based on data, not guessing.
What data do I trust? — Platform backend stats, Hongguo real-time trending chart, my own episode completion rates after publishing.
What output would I use next week? — Cross-platform summary: best genre by completion rate, best platform for new followers, what to produce next.
What failure mode is embarrassing? — Character face changes between episodes. Robotic voiceover. No hook in first 3 seconds. Content taken down for violation.
What tiny sample proves this works? — 1 theme → 10-step pipeline → 1 episode → 72-hour data. If completion ≥ 30%, scale to 10 episodes.

must serial⚠ most critical

Claude · 20 min
must serial

Every episode must have a hook AND a cliffhanger. Table format. Each cell under 20 words.

4

GPT · 15 min/episode
parallelautomatable

200-word script · 8-shot storyboard table · 8 Seedance image prompts per episode. Each prompt must include character reference + forbidden changes.

5
Video Generation
Seedance 2.0 · 60 min/episode
parallelautomatable

Full Reference Mode. Always attach character pack. Delete and regenerate any clip where face has drifted. Save to data/enriched/episode-N/clips/

6
Voiceover Generation
ElevenLab · 15 min/episode
parallelapproved script only

Only after checkpoint 4 passed. Costs quota. Add breathing + micro-pauses before power moments. English for YouTube · Mandarin for Hongguo.

7
Final Edit
CapCut / 剪映 · 30 min/episode
human review

Sync audio (≤0.5s offset) · Subtitles · BGM · Sound effects at power moments · Transitions ≤0.3s · Final runtime: 60–90s (Hongguo) or 3–5min (YouTube)

8
Thumbnail + Title A/B
Canva + GPT · 20 min/episode
parallel w/ Step 7

3 title options per platform · 2 thumbnail concepts (A/B) · YouTube: add [MULTI SUB] for Latin America + Southeast Asia reach

9
Publish + 72-Hour Data Collection
Hongguo → YouTube → X · 10 min
runner automation

Log to database · Set 24h + 72h reminders · Pass if completion ≥ 30% → scale · Fail → diagnose before producing more

CLAUDE.md

Read this file at the start of every Claude Code session. Do not override these rules without explicit approval.

Tool Roles — No Overlap

ClaudeStory structure, character arcs, plot density, episode outlinesStoryboards, image prompts
GPTStoryboards, shot lists, image prompts, production tasksLong-form narrative
ElevenLabVoiceover only — after script is lockedAnything before approval
Seedance 2.0Video generation, character consistencyFinal editing
CanvaThumbnails, character cards, visual templatesVideo production
CapCut / 剪映Final edit, subtitles, BGM, transitionsStory or script work

★ VideoScribe removed from pipeline. Whiteboard animation tool — not suitable for anime drama.

The Hook Rules (Non-Negotiable)

0–3 seconds: Conflict, humiliation, danger, secret, or identity reversal — one of these, immediately. No world-building.
Every 10 seconds: A new power moment, reversal, or revelation.
Final 5 seconds: Cliffhanger or unanswered question. Never resolve the episode.

Character Consistency Rules

Bad Prompt rejected
"A cool female lead walks into the ruins."
Good Prompt approved
"Using Character Reference A. Same face, same black hair, same white robe,
same jade pendant. Shot: medium, female lead steps through collapsed gate,
slow push-in, cold light, tense atmosphere.
Forbidden: change face shape, change hair color, add accessories."

Failure Modes

  • Character face changes between episodes — audience drops immediately
  • Voiceover sounds robotic with no emotional pauses — completion rate collapses
  • First 3 seconds have no conflict — algorithm won't distribute
  • Episode ends without cliffhanger — no next-episode pull
  • Content flagged for copyright or policy violation — taken down
  • Audio and video out of sync by more than 0.5 seconds

04

schema.sql

SQLite database for Template C reporting pipeline. Six tables covering episodes, publishes, 24h/72h metrics, character registry, weekly summaries, and production log.

episodes master

episode_number, series_name, title_cn, title_en, genre, sub_genre, production_cost, production_hours, character_consistent

publishes per platform

episode_id, platform, format, publish_date, thumbnail_ver (A/B), title_used, tags, status, url

metrics_24h data

views, completion_rate, click_rate, saves, shares, comments, new_followers, heat_score

metrics_72h go/no-go

views, completion_rate, click_rate, saves, revenue_estimate, pass_threshold (1 if ≥30%)

characters registry

name, description, face/hair/eyes/outfit, do_not_change rules, reference_images paths, locked flag

weekly_summary report

best_platform, best_genre, avg_completion_rate, total_revenue_est, total_cost, roi_ratio, key_insight, next_week_action

-- Go/No-Go query after each episode
SELECT e.title_en, p.platform, p.publish_date,
       m.views, m.completion_rate, m.pass_threshold
FROM episodes e
JOIN publishes p ON e.id = p.episode_id
JOIN metrics_72h m ON p.id = m.publish_id
ORDER BY p.publish_date DESC LIMIT 10;
05

Runner Integration

Runner = Operations layer. Claude Code handles production (scripts, video). Runner handles operations (data collection, scheduling, monitoring, reporting). They do not overlap.
01
Daily Content Briefing
Every morning at 8:30am. Pulls Google Sheets performance data, flags best/worst completion rates, checks Gmail for platform notifications, tells you what to produce today in one sentence.
Automation · Daily
02
Post-Publish Data Collection
Triggered after each publish. Logs views, completion rate, CTR to Sheets at 24h. If completion < 30%, auto-drafts Gmail diagnosis (hook too slow? thumbnail underperformed?).
Workflow · Triggered
03
Weekly Performance Report
Every Monday 9am. Pulls last 7 days from Sheets, ranks by completion rate, drafts Google Doc report, sends Gmail with link + one key insight. Under 300 words. Read in 5 minutes.
Automation · Monday
04
Heartbeat Monitoring
Background always-on. Notifies immediately: platform policy emails, copyright flags, completion rate drops below 20%, 3+ days without publishing.
Heartbeat · Always-on
05
AI Anime Workspace
Isolated workspace in Runner: Gmail + Google Sheets + Google Drive + Google Docs. Morning briefing stays separate from personal. Memory scoped to this project only.
Workspace · Isolated

Runner Memory — Set Once
